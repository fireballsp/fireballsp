# Project 01 — Regional Cloud Migration with Security & Governance

## One-liner
Designed and led a **risk-managed regional migration** of business-critical workloads into cloud with **secure-by-default architecture**, governance alignment, and operational readiness.

## Problem
Legacy workloads required cloud migration with strict constraints: uptime, data protection, identity dependencies, and auditable controls.

## My role
Cloud architect owning **target architecture**, security posture, migration approach, and cross-team alignment.

## Architecture & security highlights (what matters)
- Defined **landing zone usage**, environment separation, and least-privilege access boundaries.
- Implemented **network segmentation** and controlled ingress/egress design.
- Ensured **encryption, logging, and auditability** were part of Day-1 design.
- Planned cutover steps with safety checks and rollback strategy.

## Key decisions
- Prefer managed capabilities where they reduced security and ops burden.
- Treat governance as a delivery enabler (clear ownership, standards, documentation).

## Outcome
- Migration delivered with controlled risk and improved long-term operability.

## Stack
Azure • EntraID • Network segmentation • Central logging • IaC patterns • Zero Trust
