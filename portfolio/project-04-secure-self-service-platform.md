# Project 04 — Secure Self-Service Cloud Platform (Guardrails)

## One-liner
Enabled rapid environment provisioning through self-service while maintaining **security, lifecycle control, and governance visibility**.

## Problem
Teams needed speed; central teams needed consistent controls, visibility, and cost/risk containment.

## My role
Architecture + security enablement: define guardrails, permission boundaries, lifecycle patterns, and reusable documentation.

## Architecture & security highlights
- **Least-privilege** vending model and role boundaries.
- Automated **lifecycle controls** and safe defaults.
- Defined observability baseline for security visibility.

## Key decisions
- Security controls built into defaults, not retrofitted.
- Clear separation between platform responsibilities and consumer responsibilities.

## Outcome
- Increased self-service adoption with lower shadow IT risk and improved baseline security.

## Stack
Terraform • GitHub Actions • IAM boundaries • Policy guardrails • Lifecycle controls
