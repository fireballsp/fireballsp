# Project 05 — Hybrid Networking & DNS Security Architecture

## One-liner
Defined secure and reliable hybrid DNS resolution patterns to prevent outages and reduce misconfiguration risk in hybrid cloud deployments.

## Problem
Hybrid workloads experienced resolution failures and insecure DNS configurations impacting reliability and security.

## My role
Led design review and pattern definition; documented reusable approach for future workloads.

## Architecture & security highlights
- DNS resolution strategy aligned with identity dependencies.
- Prevented drift by favoring infrastructure-managed patterns over manual VM changes.
- Reduced exposure via controlled routing and scoped resolution.

## Key decisions
- Prefer IaC-managed DNS/resolution patterns.
- Document operational ownership and troubleshooting paths.

## Outcome
- Improved reliability and reduced security misconfiguration risk.

## Stack
Azure • Hybrid networking • Private DNS resolver patterns • IAM/Identity dependencies • IaC
