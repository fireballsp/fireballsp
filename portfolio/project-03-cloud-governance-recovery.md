# Project 03 — Cloud Governance & Security Recovery (Legacy Environment)

## One-liner
Recovered a legacy cloud environment with governance gaps by reducing exposure, tightening identity, and establishing enforceable baselines.

## Problem
Environment created without governance: identity sprawl, inconsistent boundaries, unclear ownership, and elevated exposure risk.

## My role
Architecture-led remediation: assess, risk-rank, define remediation plan, and guide execution.

## Architecture & security highlights
- Identified high-risk issues: **excess privilege, public exposure, missing telemetry, unclear ownership**.
- Defined **risk-ranked remediation backlog** and incremental execution approach.
- Restored governance via role boundaries, network segmentation, and logging baselines.

## Key decisions
- Incremental recovery to avoid destabilizing services.
- Fix highest-risk attack paths first (identity + exposure).

## Outcome
- Reduced attack surface and improved audit posture without disruptive rebuild.

## Stack
IAM hardening • Network exposure remediation • Central logging • Governance baselines
